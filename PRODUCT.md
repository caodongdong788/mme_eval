# Product

## Register

product

## Users

内部使用者，非对外公开产品：

- **评测工程师 / 算法工程师**：在浏览器里发起评测、调判分模型与综合分上线阈值、对历史 run 做离线重判 / 断点续跑，定位失败用例。
- **产品 / 质量负责人**：看通过率与跨 run 趋势，判断被测医疗 chatbot 的整体质量与回归。
- **医学审核人员（HITL）**：在人工审核队列里逐条核查需人审 / 上线失败 / 高 LLM 离散度（`score_dispersion≥0.5`）/ 红旗用例，给出同意 / 推翻裁定（只读旁路，不回写判分）；可用校准 API 上传人审表算一致率。

使用语境：日常在工位浏览器里反复操作的「内行工具」——高频、面向数据、需要专注分析而非被界面打扰。核心任务是**快速判断一个 AI 医疗咨询 / Agent chatbot 是否安全、合规、好用，并能追溯每一处判分**。

## Product Purpose

MME · Agent 评测平台：面向 AI 医疗咨询 / Agent chatbot 的**内部自动化评测平台**（极简 SaaS 形态）。把 YAML 定义的对话用例 → 并发调用被测 chatbot → 三层 judge（硬门槛 + 规则 + LLM-as-Judge）打分 → 产出报告 / 趋势看板 / 用例明细 / 人审队列。

存在意义：让医疗 chatbot 的质量评测**可复现、可追溯、可协作**。成功的样子是——评测工程师打开看板即可一眼看出通过率、失败样本、判分口径变化与安全红线，并能顺着血缘（重判 / 续跑 parent、判分 fingerprint、留痕链路）追到每条用例的判分依据。

## Brand Personality

三词：**数据驱动 · 高效克制 · 工具感专业**。

- 语气：精确、克制、专业；陈述事实与证据，不煽情、不营销话术。
- 情绪目标：让专业用户**信任数据、专注分析、零干扰**——界面是可靠的仪器，不是展品。

## Anti-references

明确不要长成这些样子：

- **花哨营销落地页**：大面积渐变、插画、滚动动效轰炸、首屏 hero 叙事。
- **老旧医院信息系统**：拥挤死板的表格、灰蓝配色、无层级的字段堆砌。
- **玩具化 / 卡通**：过度圆角、糖果色、拟物花哨。
- **塞满图表炫技的「大屏」式仪表盘**：为视觉冲击牺牲可读性的密集图表墙；本平台密度服务于分析，不为炫酷。

## Design Principles

1. **数据是主角，界面退后**：让分数、证据、失败标签、判据自己说话，框架 chrome 保持克制。
2. **安全红线一眼可辨**：HardGate / 红旗 / 上线失败必须在视觉上立刻可读，绝不淹没在表格行里。
3. **可追溯优于好看**：判分口径变化、run 血缘（重判 / 续跑 parent）、留痕 / 追踪链路要能顺藤摸瓜，胜过任何装饰。
4. **高密度但有呼吸**：面向高频内行，信息密度要够；靠层级与留白组织，而非把屏幕塞满。
5. **为反复使用的内行设计**：优先键盘可达、筛选、批量与一致的操作位；不为首次访客的花哨体验买单。

## Accessibility & Inclusion

无强制合规要求（内部工具），以合理默认为底线：

- 文本与关键状态对比度向 **WCAG AA** 看齐。
- 状态不只靠颜色：通过 / 失败 / 稳定性 / 红旗等同时带文字或标签（兼顾色觉差异）。
- 尊重系统 `prefers-reduced-motion`，动效可降级，不依赖动画传达关键信息。
